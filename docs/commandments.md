# The Ten Commandments of the Pod

**I. Everything runs through an agentic session.**
Every piece of work — code, research, debugging, documentation — happens inside a coding agent session. This is non-negotiable. Sessions are the Pod's unit of observability: if it wasn't in a session, it didn't happen, and we can't learn from it.

**II. Teach, don't just do.**
If you solve something twice, make it a skill. Knowledge belongs in the system, not in someone's head. The goal is that any Pod member — or their agent — can pick up where you left off.

**III. The human steers, the agent rows.**
Agents work autonomously, but humans define direction, review output, and carry responsibility. No code ships without a human decision. The agent is the engine, not the pilot.

**IV. Share the replay.**
Session transcripts are shared. Others can read how a problem was solved — learning by replay, not by retelling. This turns individual experience into team knowledge.

**V. Pull collectively.**
No individual ticket assignments. The Pod pulls work from the backlog as a unit. Who does it matters less than that it gets done well.

**VI. Quality over speed.**
AI amplifies throughput, never at the expense of code quality. Review standards stay the same or rise. A fast PR that introduces regressions is a net negative.

**VII. Fail fast, share the scar.**
Failed approaches are documented openly. A dead end that's shared saves the next person from walking it. Silent workarounds are the enemy.

**VIII. Measure before you optimize.**
No process change without a baseline. Measure first, change second, measure again. Gut feelings are hypotheses, not conclusions.

**IX. One backlog, one team.**
The Pod is not a silo. Same backlog, same ceremonies, same accountability as the rest of the team. The Pod changes *how* work is done, not *where* it belongs.

**X. Default to transparent.**
Decisions, approaches, and results are public. No magic behind closed doors. If someone asks "how did you do that?", the answer is a link, not a story.
