# S3 credentials, access, and AgentsView roots

Use boto3's standard credential chain. Keep secrets out of
`~/.config/rhdh-skill/config.json`, repository files, commands printed to the
user, and model context.

## Recommended environment

Load these variables through a secret-aware environment manager such as
direnv. Do not read or display the file containing their values.

```bash
export S3_BUCKET=team-agent-sessions
export S3_REGION=eu-central-1
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
```

Temporary credentials may also set `AWS_SESSION_TOKEN`. Alternatively,
configure a standard AWS profile and pass only its non-secret name with
`config init --profile PROFILE`.

## Required IAM access

| Action | Resource | Used for |
|---|---|---|
| `s3:ListBucket` | bucket ARN | setup checks, root discovery, AgentsView polling |
| `s3:GetObject` | bucket object ARN | AgentsView parsing |
| `s3:PutObject` | bucket object ARN | SessionEnd and explicit sharing |

Deletion is not required. Scope permissions to the configured bucket and
optional prefix.

## Capability gates

Run these commands after configuration and after rotating credentials:

```bash
"$FS" s3 check
"$FS" s3 roots
"$FS" s3 agentsview-config --data-dir "$HOME/.local/share/fullsend-agentsview"
```

`s3 check` is read-only and proves the active boto3 identity can list the
bucket. `s3 roots` discovers uploaded machine prefixes and prints complete
AgentsView roots such as:

```text
s3://team-agent-sessions/alice-laptop/raw/claude
```

An empty root list means access works but no compatible Claude transcript has
been uploaded yet; run an explicit share and retry.

`s3 agentsview-config` atomically updates only `claude_project_dirs` in the
private runtime `config.toml`, preserving generated secrets and unrelated
settings.

For archives created by the legacy exporter, preview and then repair only its
generated first-line metadata so AgentsView groups human sessions by repository
instead of `<machine>_<repository>`:

```bash
"$FS" s3 repair-project-metadata
"$FS" s3 repair-project-metadata --apply
```

The repair leaves the original Claude transcript lines untouched.

## Error handling

- Missing bucket/region: rerun `config init` with environment variables or
  explicit flags.
- Missing boto3: install the project with its `s3` extra before retrying.
- `AccessDenied` on list: correct `s3:ListBucket`; changing SDKs or profiles
  containing the same key does not change IAM permissions.
- Upload failure: correct `s3:PutObject`; manual `share` returns non-zero.
- AgentsView sees no sessions: verify `s3:GetObject`, then compare configured
  roots exactly with `s3 roots` output.
